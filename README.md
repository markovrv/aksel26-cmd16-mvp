# 🚆 Маршрутка — Промышленный квест по профориентации

**Маршрутка** — это интерактивная веб-платформа для знакомства школьников и студентов с промышленными предприятиями и профессиями. Пользователь проходит виртуальный маршрут по 6 реальным заводам, выполняет задания, выбирает трек развития (предприниматель / специалист) и формирует портфолио.

---

## 🎯 Возможности

### Для ученика
- **Регистрация / вход** — создание профиля с указанием школы и категории
- **Диагностика** — 7 вопросов для определения трека (предприниматель / специалист)
- **Маршрут** — 6 станций-предприятий, на каждой 4 блока с мини-заданиями
- **3D-игры** — погружение в профессии (прораб, энергетик, инспектор КК) через Three.js
- **Маркет** — магазин аватаров, достижения
- **Профиль** — баллы, уровень, статистика прохождения, заявки на экскурсии
- **Финальный результат** — готовый проект/траектория после всех станций
- **Портфолио** — PDF-экспорт достижений

### Для педагога
- **Кабинет педагога** — просмотр учеников своего класса
- **Фильтрация** по классу
- **Статистика** по каждому предприятию
- **Экспорт PDF** — ведомость учеников

### Для администратора
- **Панель администрирования** — управление всеми сущностями системы через интерфейс
- CRUD пользователей, предприятий, блоков станций, вопросов, достижений, задач, итоговых результатов
- Настройка баллов, порогов уровней, игровых конфигураций
- Авто-генерация блоков станций для новых предприятий
- Экспорт данных

## 🚀 Быстрый старт

### 1. Требования
- Node.js ≥ 20
- npm
- Git

### 2. Установка

```bash
git clone https://github.com/markovrv/aksel26-cmd16-mvp.git
cd 16
npm install        # установит зависимости client + server
```

### 3. Настройка окружения

```bash
cp server/.env.example server/.env
# Отредактируйте при необходимости
```

По умолчанию используется:
- Порт: `4173`
- Админ: `admin@marshrutka.ru` / `admin123`

### 4. Миграция и сидирование БД

```bash
npm run db:migrate    # Создание таблиц
npm run db:seed       # Заполнение данными
```

### 5. Запуск

```bash
npm run dev           # Сервер на порту 4173
npm run dev:client    # Vite-девсервер на порту 5173 (отдельный терминал)
```

### 6. Открыть в браузере
- **http://localhost:5173** — фронтенд (через Vite proxy на /api → 4173)
- **http://localhost:4173** — production-режим (если собран)

## 🧑‍🏫 Инструкция для пользователей

### Ученик
1. **Главная** → нажмите **Начать маршрут**
2. **Регистрация** → укажите имя, email, пароль, школу
3. **Диагностика** → ответьте на 7 вопросов (определится трек)
4. **Аватар** → настройте внешность
5. **Маршрут** → проходите предприятия по порядку
6. На каждом предприятии — **4 блока** с информацией и вопросами
7. После всех 6 станций — **финальный результат**
8. **Профиль** — отслеживайте прогресс

### Педагог
1. В хедере нажмите **🔑 Педагогу**
2. Введите пароль педагога или используйте демо-вход
3. Откроется **Кабинет педагога** со списком учеников вашего класса
4. Можно фильтровать по классу, смотреть статистику, экспортировать PDF

### Администратор
1. В хедере нажмите **⚙️ Админ**
2. Введите `admin@marshrutka.ru` / `admin123` или нажмите **Демо-вход**
3. Откроется **Панель администратора** с вкладками:
   - 👥 **Пользователи** — поиск, просмотр, редактирование, удаление
   - 🏭 **Предприятия** — создание, редактирование, удаление
   - 🧩 **Блоки станций** — управление блоками, авто-генерация
   - ❓ **Вопросы** — управление вопросами диагностики и ответами к ним
   - 🏆 **Достижения** — CRUD достижений
   - 📋 **Задачи** — управление игровыми задачами
   - 📊 **Итоговые результаты** — финальные описания для треков
   - ⚙️ **Баллы** — настройка системы скоринга
   - 📈 **Уровни** — пороги уровней
   - 🎮 **Игра** — работники и строки документации для 3D-задач
   - 📤 **Экспорт** — выгрузка данных

## 📡 API (кратко)

### Публичные эндпоинты
- `POST /api/auth/register` — регистрация
- `POST /api/auth/login` — вход
- `POST /api/auth/refresh` — обновление токена
- `GET /api/config/*` — публичные конфиги
- `GET /api/statistics/*` — публичная статистика

### Защищённые эндпоинты (JWT)
- `GET|PUT /api/profile` — профиль
- `GET /api/companies` — предприятия
- `GET /api/companies/:id/pieces` — блоки станции
- `POST /api/station-pieces/:id/complete` — завершить блок
- `POST /api/questions/submit` — отправить ответы диагностики
- `POST /api/tasks/:id/solve` — решение задачи
- `GET /api/final-result` — финальный результат
- `GET /api/teacher/students` — ученики (педагог)

### Админ-эндпоинты (JWT + admin)
- `GET|POST|PUT|DELETE /api/admin/*` — полный CRUD всех сущностей

Полный список — в [docs/backend-design.md](docs/backend-design.md)
