#### C1. Портирование игрового кода в React

Перенеси код из `/game/src/` в `client/src/game/`, адаптируя под React Three Fiber:

| Исходный файл (`/game/src/`) | Целевой компонент (`client/src/game/`) |
|---|---|---|

| `core/GameStateManager.js` | Контекст `GameStateContext` (уже создан на этапе A) |
| `core/UIManager.js` | React-компоненты HUD, модалов, тостов |
| `core/AnimationManager.js` | `managers/useAnimations.js` — хук с GSAP-анимациями |
| `scenes/EntryScreen.js` | Не портировать — экран входа заменён на React-модалы |
| `scenes/MainMenu.js` | Не портировать — выбор профессии происходит через карту станций |
| `scenes/ProfessionIntro.js` | `scenes/ProfessionIntroR3F.jsx` — интро профессии с 3D-превью |
| `scenes/ConstructionSite.js` | `scenes/ConstructionSiteR3F.jsx` |
| `scenes/DispatchRoom.js` | `scenes/DispatchRoomR3F.jsx` |
| `scenes/InspectionSite.js` | `scenes/InspectionSiteR3F.jsx` |
| `scenes/ResultScreen.js` | `scenes/ResultScreenR3F.jsx` — конфетти на экране результатов |
| `tasks/ForemanTasks.js` | `tasks/ForemanTasks.jsx` — логика задач + UI-оверлеи |
| `tasks/EngineerTasks.js` | `tasks/EngineerTasks.jsx` |
| `tasks/InspectorTasks.js` | `tasks/InspectorTasks.jsx` |
| `utils/PDFExporter.js` | `utils/PDFExporter.js` — адаптировать под общий формат портфолио |

**Правила портирования:**
- `new THREE.Scene()` → `<Canvas>` из R3F.
- `new THREE.Mesh(...)` → JSX-элементы (`<mesh>`, `<boxGeometry>`, `<meshStandardMaterial>`).
- `scene.add(...)` → вложенность JSX.
- `requestAnimationFrame` / `renderer.render` → автоматически через R3F.
- OrbitControls → `<OrbitControls>` из Drei.
- GSAP-анимации → оставить через GSAP (он работает с `useRef` и `useFrame`).
- Обработчики кликов по 3D-объектам → `onClick` на mesh + `useThree()` для raycasting'а.
- Canvas-текстуры (мониторы в диспетчерской) → через `useMemo` + `new THREE.CanvasTexture(...)`.

#### C2. Связь игры с платформой

Игра запускается **внутри блока 4** выбранных станций (см. маппинг выше):

1. Пользователь нажимает на блок 4 станции.
2. Открывается полноэкранный режим с 3D-сценой (скрывается остальной UI платформы, показывается HUD игры).
3. Пользователь проходит 3 задачи последовательно.
4. Результат каждой задачи (баллы, время, правильность) сохраняется в общее состояние платформы.
5. После задачи 3 — экран результатов с конфетти, затем возврат на карту станций.
6. Прогресс, баллы и достижения немедленно отражаются в профиле и на карте.

#### C3. Адаптация достижений

Достижения из игры должны регистрироваться в общем списке достижений платформы. При выполнении условия достижение появляется в профиле пользователя.

#### C4. Адаптация PDF-экспорта

Переиспользуй `PDFExporter.js` из игры, адаптировав:
- Шапка: «Цифровое портфолио платформы Маршрутка».
- Данные пользователя: ФИО, возраст/категория, трек.
- Пройденные станции и профессии с баллами.
- Навыки (звёзды ⭐).
- Таблица всех задач (станция, задача, баллы, время, статус).
- Все достижения.
- Рекомендация платформы (из финального результата).
- QR-код (опционально).

Кнопка «Скачать PDF» доступна:
- В профиле (итоговое портфолио).
- На экране результатов после каждой игровой профессии.
- На финальном экране маршрута.

## Нефункциональные требования

### Адаптивность
- **Десктоп (≥1024px):** полный 3D-режим в блоке 4, OrbitControls, HUD.
- **Планшет (768–1023px):** упрощённый 3D (сниженное разрешение, меньше полигонов), HUD компактный.
- **Мобильный (<768px):** 2D-режим для всех задач (выбор варианта, drag-drop, чекбоксы) — 3D-сцена не загружается.

### Прогрессивный рендеринг
- При недоступности WebGL (проверка через `canvas.getContext('webgl2')` или `canvas.getContext('webgl')`) — блок 4 автоматически переключается в 2D-режим с кнопками, без попытки загрузить Three.js.
- Пользователь не должен видеть ошибок или пустых экранов.

### Edge cases (обязательно обработать)
- **Сброс localStorage:** при первом запуске (чистый localStorage) — экран приветствия, предложение создать профиль.
- **Повторное прохождение станции:** если все 4 блока уже пройдены — кнопки блоков активны для повторного просмотра, но баллы повторно не начисляются.
- **Возврат с игры во время прохождения:** если пользователь закрыл вкладку во время 3D-игры — прогресс незавершённых задач не сохраняется; пройденные задачи сохраняются.
- **Обновление страницы во время игры:** состояние восстанавливается из localStorage — пользователь возвращается на карту станций, пройденные задачи учтены.
- **Конфликт данных:** если структура localStorage изменилась между версиями — сброс до defaultState с предупреждением в консоли.
- **Двойной клик по заданиям:** кнопки должны блокироваться (disabled) на время обработки ответа, чтобы исключить двойное начисление баллов.
- **Таймер:** для задач с бонусом за скорость — таймер запускается при показе задачи и останавливается при ответе; бонус начисляется, если время меньше порога из game.md.
